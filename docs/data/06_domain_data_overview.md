## 4. RAG 파이프라인 활용 방안

### 4-1. Qdrant 인덱싱 전략

```
QnA 데이터 → 질문+답변 청크 → Dense 임베딩 → Qdrant
품종 메타 → 품종명+설명 청크 → Dense 임베딩 → Qdrant
```

**컬렉션 설계안**:

| 컬렉션 | 소스 | 임베딩 텍스트 | payload |
|---|---|---|---|
| `domain_qna` | QnA 6개 파일 | `질문 + 답변` | species, category, source |
| `breed_meta` | 품종 2개 파일 | `품종명 + 수의 영양학적 메타 디스크립션` | species, breed_name, group, health_keywords |

### 4-2. 챗봇 활용 시나리오

| 사용자 질문 유형 | 활용 데이터 |
|---|---|
| "우리 강아지가 초콜릿을 먹었어요" | QnA — 건강 및 질병 |
| "보더 콜리에게 맞는 사료 추천해줘" | 품종 메타 + 상품 데이터 |
| "고양이가 밥을 안 먹어요" | QnA — 영양 및 식단 |
| "고양이 모래 어떤 걸 써야 해?" | bemypet_product QnA |
| "강아지랑 비행기 타도 돼요?" | biteme_blog QnA — 여행 및 이동 |

### 4-3. 품종 기반 상품 추천 연계

```
사용자 펫 프로필 (품종)
    → 품종 메타에서 건강 특징·권장 사료 유형 조회
    → PET_HEALTH_CONCERN 태그 자동 매핑
    → Qdrant 필터링 검색으로 관련 상품 추천
```

예시: 보더 콜리 → `관절`, `고단백 활동견 사료` → 관절 영양제·고단백 사료 상품 필터링

## 5. 추가 메타데이터 제안

### QnA에 추가하면 좋은 필드

| 필드 | 값 예시 | 용도 |
|---|---|---|
| `species` | `dog` / `cat` / `both` | 종 필터링 |
| `urgency` | `emergency` / `general` | 긴급 상황 분류 |
| `age_group` | `puppy` / `adult` / `senior` | 연령대 필터링 |
| `breed_specific` | `true` / `false` | 견종 특이적 질문 여부 |

### 품종 메타에 추가하면 좋은 필드

| 필드 | 값 예시 | 용도 |
|---|---|---|
| `size` | `소형` / `중형` / `대형` | 사료 용량 필터링 |
| `activity_level` | 1~5 | 활동량 기반 칼로리 추천 |
| `health_concern_tags` | `["관절", "피부"]` | PET_HEALTH_CONCERN 직접 매핑 |
| `allergen_risk` | `["닭고기", "유제품"]` | 알레르기 주의 성분 |
| `recommended_food_type` | `["dry", "wet_can"]` | PET_FOOD_PREFERENCE 매핑 |

## 6. sLLM 파인튜닝 관점 분석

### 6-1. 데이터 규모 및 텍스트 통계

| 항목 | 수치 |
|---|---|
| 전체 QnA 건수 | 2,412쌍 |
| 질문 평균 길이 | 38자 |
| 답변 평균 길이 | 93자 |
| 답변 최대 길이 | 183자 |
| 답변 길이 분포 | ~100자(68%), 101~200자(32%) |

### 6-2. 파인튜닝 적합성 평가

| 항목 | 평가 | 비고 |
|---|---|---|
| 데이터 규모 | **보통** | Instruction tuning 최소 기준(~1,000건) 충족. Full fine-tuning엔 부족, LoRA/QLoRA는 가능 |
| 답변 품질 | **미흡** | 평균 93자로 매우 짧음. 실제 챗봇 응답 스타일(200~500자)과 괴리 |
| 도메인 커버리지 | **양호** | 건강·영양·사육·행동 4개 주요 카테고리 균형적으로 분포 |
| 한국어 품질 | **양호** | 수의학 전문 용어 포함, 참고문헌 기반 작성 |
| 포맷 변환 용이성 | **우수** | `질문→답변` 구조가 Instruction tuning 포맷으로 직접 변환 가능 |

### 6-3. 파인튜닝 vs RAG 용도 분리 권장

```
┌─────────────────────────────────────────────────────┐
│  파인튜닝 적합                                        │
│  - 도메인 용어 주입 (슬개골 탈구, 테오브로민 등)         │
│  - 반려동물 전문가 톤·스타일 학습                       │
│  - 카테고리 분류 능력 (건강/영양/행동 판별)              │
├─────────────────────────────────────────────────────┤
│  RAG 적합 (파인튜닝보다 우선)                          │
│  - 특정 질문에 대한 정확한 답변 검색                    │
│  - 품종별 영양 정보, 상품 추천 연계                     │
│  - 출처 근거 제시                                     │
└─────────────────────────────────────────────────────┘
```

**결론**: 현재 데이터만으로 파인튜닝 단독 적용은 권장하지 않음. **RAG + 프롬프트 엔지니어링**으로 우선 구현하고, 파인튜닝은 답변 스타일 개선 목적으로 보조 적용하는 것이 적합.

### 6-4. 파인튜닝 적용 시 권장 방식

**방식**: LoRA / QLoRA (전체 파라미터 대비 비용 효율적)

**학습 데이터 포맷 변환 예시**:
```json
{
  "instruction": "반려동물 전문가로서 질문에 답변해주세요.",
  "input": "강아지에게 초콜릿이 치명적인 이유는 무엇인가요?",
  "output": "초콜릿의 '테오브로민' 성분 때문입니다. 강아지는 이를 대사하는 능력이 매우 낮아 심박수 급증, 경련, 구토, 심지어 사망에 이를 수 있습니다. 특히 다크 초콜릿이 가장 위험합니다."
}
```

**답변 품질 보완 방안**:
- 현재 93자 평균 답변을 그대로 사용 시 모델이 짧은 응답만 생성할 위험 있음
- `참고사항` 컬럼 내용을 답변에 병합하여 답변 길이 보강 후 학습 권장
- 품종 메타의 `수의 영양학적 메타 디스크립션`은 길이·품질이 우수하여 파인튜닝 데이터로 가장 적합

**데이터 증강 제안**:
- 질문 패러프레이징 (동일 답변에 다양한 질문 표현 생성)
- 멀티턴 대화 형식으로 변환 (단일 QnA → 2~3턴 대화)
- 상품 추천 맥락 추가 (답변 끝에 관련 상품 카테고리 언급)

## 7. 데이터 품질 및 유의사항

- `참고사항` 컬럼에 `=IMAGE(...)` 수식이 포함된 행 존재 → 전처리 시 제거 필요
- 고양이 질의응답 1000선의 `No.` 컬럼이 float 타입 (`1.0`, `2.0`) → int 변환 필요
- 품종 메타의 `이미지` 컬럼은 전부 NaN → 별도 이미지 수집 필요 시 품종명 기반 검색
- QnA 출처가 bemypet / biteme / 자체제작 혼재 → `source` 태그로 구분 권장
