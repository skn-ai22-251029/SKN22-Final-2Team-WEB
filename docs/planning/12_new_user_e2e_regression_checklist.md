# 신규 유저 핵심 플로우 E2E 회귀 체크리스트

> 이 문서는 이슈 `#236` 대응용이다.
> 목표는 신규 유저가 처음 유입된 뒤 추천, 장바구니, 주문 완료까지 한 번에 점검할 수 있게 만드는 것이다.

---

## 1. 범위

- 소셜 로그인 후 신규 유저 진입
- 프로필 초기 설정
- 반려동물 등록 또는 예비 집사 분기
- 챗봇 추천 생성과 우측 패널 갱신
- 장바구니 추가와 주문 완료 페이지 진입

---

## 2. 사전 조건

- 신규 소셜 계정 또는 프로필/반려동물 정보가 없는 테스트 계정
- 상품 데이터가 적재된 상태
- 채팅 추천이 가능한 FastAPI 연동 상태
- 주문 API와 장바구니 API가 동작하는 상태

권장 브라우저:

- Chrome 최신 버전
- 모바일 해상도 확인이 필요하면 DevTools iPhone 14 Pro 또는 390px 폭 기준 추가 점검

## 4. 시나리오 A. 소셜 로그인 후 프로필 진입

### 목표

- 신규 유저가 소셜 로그인 후 초기 설정 화면으로 자연스럽게 유도되는지 확인한다.

### 체크리스트

- [ ] 로그인 화면에서 Google / Kakao / Naver 진입 버튼이 보인다.
- [ ] `/auth/{provider}/start/` 진입 시 OAuth 인증 화면으로 이동한다.
- [ ] 콜백 완료 후 신규 유저는 `/profile/?setup=1`로 이동한다.
- [ ] 프로필 초기 설정 화면에서 기본 정보 입력 UI가 보인다.
- [ ] 프로필 저장 후 연락처, 주소, 결제수단 등 기본 값이 정상 저장된다.

### 실패 시 확인 포인트

- 소셜 콜백 redirect 위치
- 세션 유지 여부
- 신규 유저 강제 setup 플래그 처리

---

## 5. 시나리오 B. 반려동물 등록 플로우

### 목표

- 신규 유저가 반려동물 온보딩을 완료하고 챗봇 진입 가능한 상태가 되는지 확인한다.

### 체크리스트

- [ ] `/pets/add/`에서 강아지 / 고양이 / 예비 집사 선택지가 보인다.
- [ ] 반려동물 분기 선택 시 `/pets/add/details/`로 이동한다.
- [ ] 이름, 성별, 나이 등 필수값 검증이 동작한다.
- [ ] `/pets/add/health/`에서 건강 관심사, 알레르기, 선호 형태, 예산대 입력이 가능하다.
- [ ] 저장 완료 후 챗봇 진입 또는 메인 진입이 정상 동작한다.

### 실패 시 확인 포인트

- 단계별 세션/폼 데이터 유지
- 필수값 검증 문구
- 저장 후 redirect 경로

---

## 6. 시나리오 C. 예비 집사 분기

### 목표

- 반려동물이 없는 신규 유저도 예비 집사 프로필로 챗봇을 사용할 수 있는지 확인한다.

### 체크리스트

- [ ] `/pets/add/`에서 예비 집사를 선택할 수 있다.
- [ ] 예비 집사 전용 입력 화면이 노출된다.
- [ ] 저장 후 챗봇 진입 시 프로필 영역에 예비 집사 상태가 반영된다.
- [ ] 반려동물 미등록 상태에서도 채팅 페이지 접근이 차단되지 않는다.

### 실패 시 확인 포인트

- FuturePetProfile 저장 여부
- 챗봇 진입 guard 조건

---

## 7. 시나리오 D. 채팅 추천과 우측 패널 갱신

### 목표

- 신규 유저가 첫 추천 질문을 던졌을 때 응답과 상품 패널이 함께 갱신되는지 확인한다.

### 체크리스트

- [ ] `/chat/` 진입 후 입력창이 활성화되어 있다.
- [ ] 추천 질문 칩 또는 직접 입력으로 첫 질문 전송이 가능하다.
- [ ] 응답이 스트리밍 또는 단계적 렌더링으로 표시된다.
- [ ] 우측 패널에 추천 상품 카드가 렌더링된다.
- [ ] 우측 패널 상품 이미지, 가격, 평점, 리뷰 수가 비정상 없이 보인다.
- [ ] 다른 질문을 한 번 더 보내면 우측 패널 상품 목록이 최신 응답 기준으로 갱신된다.

### 실패 시 확인 포인트

- Django -> FastAPI 프록시 응답
- 세션 ID 생성 / 유지
- 추천 상품 payload 존재 여부

---

## 8. 시나리오 E. 장바구니와 주문 완료

### 목표

- 추천 상품에서 장바구니로 이어지고 주문 완료 페이지까지 도달하는지 확인한다.

### 체크리스트

- [ ] 추천 상품 카드의 `+` 버튼으로 장바구니에 담을 수 있다.
- [ ] `/products/` 또는 장바구니 탭에서 담긴 상품이 보인다.
- [ ] 수량 변경과 삭제가 정상 동작한다.
- [ ] 주문하기 버튼으로 checkout 흐름에 진입할 수 있다.
- [ ] 결제수단 선택과 배송 정보 검증이 정상 동작한다.
- [ ] 주문 완료 후 `/orders/complete/<order_id>/` 페이지가 열린다.
- [ ] 완료 페이지에 주문 번호, 수령인, 결제수단, 금액 정보가 보인다.
- [ ] 챗봇으로 돌아가기 또는 후속 진입 경로가 깨지지 않는다.

### 실패 시 확인 포인트

- 장바구니 API 응답
- 주문 생성 API 응답
- 주문 완료 payload 직렬화

---

## 9. 최소 회귀 기준

아래 4개가 모두 통과하면 신규 유저 시연 플로우의 최소 회귀 기준을 통과한 것으로 본다.

- [ ] 소셜 로그인 후 `/profile/?setup=1` 진입
- [ ] 반려동물 또는 예비 집사 프로필 저장 성공
- [ ] 채팅 추천 응답과 우측 패널 상품 갱신 성공
- [ ] 장바구니 추가 후 주문 완료 페이지 진입 성공

---

## 10. 버그 기록 규칙

- 재현 경로를 route 단위로 남긴다.
- 실패 화면은 스크린샷과 함께 남긴다.
- 데이터 문제인지 UI 문제인지 먼저 분류한다.
- 추천 장애가 의심되면 `product.pet_type/category/subcategory`와 추천 payload를 함께 확인한다.
