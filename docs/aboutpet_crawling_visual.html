크롤링 수집 흐름
1
카테고리 정의
수집할 카테고리 코드(cateCdL, cateCdM, dispClsfNo) 목록 작성. 강아지/고양이 전 카테고리 대상.
2
상품 목록 수집 POST /shop/getScateGoodsList
page=1, 2, 3... 반복 호출 (rows=20). 응답 스크립트에서 총 상품 수(goodsCount) 확인 후 종료. 각 상품의 goodsId 수집.
3
상품 상세 수집 GET /goods/getGoodsDetail
goodsId별로 상세 페이지 호출. 상품 필수 정보 테이블(원산지, 제조사 등) 파싱.
4
평점 분포 수집 POST /goods/getGoodsCommentScore
평균 평점 + 별점별 비율(1~5점) 수집.
5
후기 목록 수집 POST /goods/getGoodsEntireCommentList
페이지 반복 호출로 전체 후기 수집. 후기 내용, 별점, 작성일, 닉네임 등 파싱.
📡

주의 사항
  • 📄
    무한 스크롤이지만 API는 페이지 파라미터 지원
    UI는 무한 스크롤이나 내부 API는 page 파라미터를 지원하므로 순차 호출로 전체 수집 가능.
  • ⭐
    평점 스케일 혼용 주의
    목록 API의 data-goodsstarsavgcnt는 10점 만점, 상세 페이지 .starpoint .point는 5점 만점. 정규화 필요.
  • ⏱️
    요청 간 딜레이 설정 권장
    서버 부하 방지 및 차단 우회를 위해 요청 사이에 1~2초 간격 설정 권장.
  • 🖼️
    영양 성분·원재료는 이미지로 제공될 수 있음
    일부 상품 상세 설명은 텍스트가 아닌 이미지로 렌더링됨. OCR 처리가 필요할 수 있음.
  • 🔐
    후기는 로그인 없이 조회 가능
    후기 목록 조회 API는 비로그인 상태에서도 호출 가능. 단, 후기 작성은 구매자만 가능.
